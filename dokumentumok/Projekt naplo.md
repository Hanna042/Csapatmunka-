# Projekt napló

## Csapattagok és szerepkörök

A projektet hárman készítjük:

### Én

Feladataim:

* termékek megjelenítése a weboldalon (get.js)
* termék módosítás funkció megvalósítása
* Git repository létrehozása
* termék módosítása (modositas.js, modositas.html)

### Balázs

Feladatai:

* dokumentáció elkészítése
* kód kommentelése
* Trello Kanban rendszer kezelése
* kosár funkció (kosar.js, kosar.test.js)

### Kristóf

Feladatai:

* a JavaScript további részeinek elkészítése
* Git repository kezelése

---

## Feladatok (Backlog)

* Termékek lekérése API-ból
* Terméklista megjelenítése
* Kosár funkció (hozzáadás, törlés, mennyiség módosítás)
* Kosár mentése
* Termék módosítás
* Navigáció oldalak között
* Hibakezelés
* Tesztelés
* Dokumentáció
* Projektmenedzsment (Trello)

---

## Sprint leírások

### 1. sprint

Ebben a sprintben az alap funkciók kialakításán dolgozunk:

* Projekt struktúra kialakítása
* Git repository létrehozása és használata
* API kapcsolat kialakítása
* Termékek megjelenítése a főoldalon
* Kosár funkció alapjainak megvalósítása
* JavaScript logika nagy részének elkészítése

---

## Daily standupok

Minden nap reggel tartunk rövid megbeszélést, ahol átbeszéljük:

* ki min dolgozott előző nap
* milyen problémák merültek fel
* ki min fog dolgozni aznap

---

## 2. sprint

A következő sprintben a cél:

* Termék módosítás funkció befejezése
* Kosár funkció teljesítése és finomítása
* Tesztelés és hibajavítás
* UI javítása
* Dokumentáció véglegesítése

---

## Tanulságok

Bár még csak az első sprintnél tartunk, már most is vannak tapasztalatok:

* A feladatok előzetes felosztása segített elkerülni az átfedéseket
* A Git használata fontos, de oda kell figyelni a közös munkára
* A napi standupok segítenek abban, hogy mindenki tisztában legyen a projekt állapotával
* A Trello rendszer jól átláthatóvá teszi a feladatokat

### Saját tapasztalataim:

* A termékmegjelenítés és API kezelés során fejlődtem JavaScriptben
* A Git repository kezelése gyakorlati tapasztalatot adott a verziókezelésben

---

Összességében a projekt jó ütemben halad, és az első sprint céljait nagyrészt sikerült teljesíteni.
